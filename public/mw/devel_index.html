Developer Tests

These are code develment tests and are not necessarily intended to be good examples.

simple world
Loads a very simple model and that's about it. A sanity check.
move via x3d data attribute
Animates a model by adding extra javaScript code via HTML interpreting "data-" attributes as files to load. At this point it's playing with the "data-" attributes and may or may not become a Mirror Worlds common method.
example move 2
Like the above, plus loads the moving model twice. This may not be the best way to do this kind of thing, but it's a good test that show how the two animations act independently.
gnome and moving plane
Moving a different part of the model just as a test.
2 client connections in one browser client
Run another server via: 
./mw_server --http_port=8881 --https_port=8883
and then click to make 2 connections with one web client
Source and sink as separate files
This is the first test of the Source/Sink architecture. Open these two links in separate windows or tabs: viewpoint source writes a veiwpoint as a position and rotation as you navigate through the scene viewpoint sink reads and sets the viewpoint
simple avators using single javaScript file without nesting
This uses a code style that may not be the best.
simple viewpoint avators
This may be a better way to make viewpoint avatars. Uses less variables, keeping the state needed in the function callback context. Shows how javaScript is pretty amazing by storing callback functions with the context at the time of setting the callbacks. This is the best explanation I could find at this time: Closures
list avatars
Lists the avatars in /mw/avatars. See the avatar X3D files in the browser debug console.
